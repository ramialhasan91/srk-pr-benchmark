# SRK / Peng–Robinson saturation-property benchmark

Reproducible benchmark of the SRK and Peng–Robinson (PR) cubic equations of state — with both the classical Soave and the generalized Twu (1995) α-functions — plus translated-PR variants (a three-rung ladder: zero-data Rackett-anchored, fitted at Tr = 0.70, and critical-matched), against reference multiparameter equations of state (via CoolProp) for 32 pure fluids: saturation pressure, saturated-liquid density, saturated-vapor density, and enthalpy of vaporization, 1280 state points, Tr = 0.50–0.985.
Includes two internal consistency checks (volume-translation invariance verified to 1e-14; departure-function vs Clapeyron enthalpy of vaporization verified to 6e-7 at every point) and a sensitivity sweep of the translation anchoring temperature.

Companion code and data for the manuscript *"How accurate are the SRK and Peng–Robinson equations of state for saturation properties? A reproducible benchmark against reference equations of state for 32 fluids."*

## Repository structure

```
.
├── code/                       Analysis pipeline (run in the order below)
│   ├── eos.py                  Cubic EoS core: SRK/PR with Soave and generalized-Twu
│   │                           α-functions (analytic derivatives), fugacity coefficients,
│   │                           saturation-pressure solver, residual enthalpy and enthalpy
│   │                           of vaporization, and the volume-translation quadrature check.
│   ├── 01_run_benchmark.py     Runs the benchmark: 4 (cubic, α) combinations × 4 properties,
│   │                           the three translated-PR density variants, the Clapeyron and
│   │                           translation-invariance checks, and the anchor sweep.
│   ├── 04_stats.py             Fluid-level cluster-bootstrap 95% CIs, exact binomial sign
│   │                           tests, and the near-critical Zc regression.
│   ├── 02_make_tables.py       Generates every LaTeX table (main text + supplementary).
│   └── 03_make_figures.py      Generates every figure as PDF (+ PNG preview).
│
├── data/                       All computed results — regenerated by the scripts.
│   ├── deviations_full.csv     Per-point results and % deviations (1280 rows).
│   ├── fluids.csv              Fluid metadata: Tc, pc, ω, Zc_ref, Z_RA, translation constants.
│   ├── anchor_sweep.csv        Pooled PR-VT density AAD vs anchoring temperature.
│   ├── key_numbers.json        Headline statistics quoted in the manuscript.
│   ├── stats.json              Bootstrap CIs, sign tests, near-critical regression.
│   └── run_metadata.json       Library versions, solver settings, convergence and check record.
│
├── figures/                    Generated figures (fig1–fig6 main text, figS1–figS3 supplementary).
├── tables/                     Generated LaTeX tables (\input by the manuscripts).
│
├── main.tex                    Main manuscript (optional; see note below).
├── supplementary.tex           Supplementary material (optional; see note below).
├── refs.bib                    Bibliography.
│
├── requirements.txt            Pinned dependency versions.
├── LICENSE                     MIT license.
└── README.md                   This file.
```

No experimental data are transcribed anywhere in the repository; every reference value is
computed at run time from the reference equations of state in CoolProp, and every number,
table, and figure is produced by the four scripts above.

> **Note on the manuscript sources.** `main.tex` and `supplementary.tex` are included only so
> that the paper can be rebuilt end-to-end; they are not required to reproduce any result. If
> you are cloning this repository before the article is published, they may be absent — the
> code, data, and generated `figures/`/`tables/` are self-contained and fully reproduce the
> analysis without them.

## Reproducing the results

Requires Python ≥ 3.11. The pinned versions used to generate the published numbers are in
`requirements.txt` (CoolProp 8.0.0, NumPy 2.4.4, pandas 3.0.2, matplotlib 3.10.8).

```bash
# 1. Install dependencies (a virtual environment is recommended)
pip install -r requirements.txt

# 2. Run the pipeline IN THIS ORDER.
#    01 writes the raw results that 04 reads; 04 writes stats.json that 02 reads.
python code/01_run_benchmark.py    # -> data/*.csv, key_numbers.json, run_metadata.json
python code/04_stats.py            # -> data/stats.json   (bootstrap CIs + sign tests)
python code/02_make_tables.py      # -> tables/*.tex
python code/03_make_figures.py     # -> figures/*.pdf (+ .png previews)
```

Each script prints a short summary as it runs; `01_run_benchmark.py` reports that all 1280
state points converged and that both internal consistency checks pass (volume-translation
invariance to ~1e-14, departure-function vs Clapeyron enthalpy of vaporization to ~6e-7).
The pipeline is deterministic (the bootstrap uses a fixed seed), so a clean run reproduces the
committed `data/`, `tables/`, and `figures/` byte-for-byte.

If the `.tex` sources are present, and with a LaTeX distribution that includes `elsarticle`:

